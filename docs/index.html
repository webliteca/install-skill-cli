Install skills with
one command

A package manager for AI assistant skills. Resolve from a registry, pin versions, and keep your team in sync with lock files.

$ npm install -g install-skill Click to copy

Why install-skill?

Manage AI assistant skills the same way you manage packages — with a registry, version pinning, and reproducible installs.

📦

Registry lookup

Install skills by name from the central skills registry. No need to remember Maven coordinates.

🔒

Lock files

Reproducible installs with .skills-versions.lock. Every contributor gets the exact same versions.

📋

Batch install

List skills in a .skills-versions file and install them all at once. Version control friendly.

🌐

Cross-platform

Native bundles for macOS, Windows, and Linux. Distributed via npm for easy global installation.

🔑

Private repos

Supports custom Maven repositories with authentication for private or enterprise skills.

⚡

Smart resolution

Only re-resolves changed entries. Unchanged skills are installed from the lock file instantly.

Quick start

Get up and running in seconds. Install a single skill or manage your full skill set with a versions file.

Single skill
# Install by registry name
$ install-skill teavm-lambda

# Pin a specific version
$ install-skill teavm-lambda@0.1.2

# Use Maven coordinates directly
$ install-skill ca.weblite:teavm-lambda-parent@0.1.2

.skills-versions

.skills-versions
# Skills for this project
teavm-lambda@0.1.2
my-other-skill@0.3.1
some-skill

List skills one per line. Supports registry names, Maven coordinates, and optional version pinning.

.skills-versions.lock

.skills-versions.lock
{
  "lockVersion": 1,
  "skills": {
    "teavm-lambda": {
      "groupId": "ca.weblite",
      "artifactId": "teavm-lambda",
      "version": "0.1.2"
    }
  }
}

Auto-generated lock file ensures reproducible installs. Commit it to version control.

Batch install
# Install all skills from .skills-versions
$ install-skill

# Install to a custom directory
$ install-skill -d ./my-skills

# Install globally
$ install-skill --global

# Force re-resolve all versions
$ install-skill --update

CLI options

All the flags you need to control where and how skills are installed.

Option Description
-d <dir> Skills installation directory (overrides --global)
-g, --global Install globally to ~/.claude/skills instead of ./.claude/skills
-r <repo> Repository URL with optional credentials: [user:pass@]repositoryUrl
-u, --update Force re-resolution of all skill versions, ignoring the lock file
-h, --help Show help message and exit
-V, --version Show version number and exit

How it works

Under the hood, install-skill uses embedded Maven to resolve and install skill artifacts.

1

Resolve coordinates

Looks up the skill name in the skills registry to find its Maven group ID, artifact ID, and available versions. You can also provide Maven coordinates directly.

2

Create temporary project

Generates a temporary Maven project with the skill listed as a dependency and the skills-jar-maven-plugin configured to run.

3

Install via Maven

Runs the embedded Maven runtime to resolve dependencies and execute the skills-jar-plugin:install goal, which extracts skill guidance files.

4

Copy to target

Moves the installed skill files to your target directory — either ./.claude/skills (local), ~/.claude/skills (global), or a custom path.

5

Update lock file

In batch mode, writes the resolved coordinates to .skills-versions.lock so subsequent installs are instant and reproducible.

Ecosystem

install-skill is part of a toolchain for creating, publishing, and installing AI assistant skills.

skills-jar-maven-plugin

Maven plugin for packaging and deploying AI assistant skills as Maven artifacts. The build-side counterpart to install-skill.

skills-registry

Central registry that maps human-readable skill names to Maven coordinates. Open a PR to register your own skills.
